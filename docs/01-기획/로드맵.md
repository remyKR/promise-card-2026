# 개발 로드맵

> 버전: v2.2 | 최종수정: 2026-05-04

---

## MVP 목표

6주 안에 서비스 출시. 기획·정책·설계 완료 후 구현 시작.

> **범위 조정 원칙**: 6주는 빠듯하다. 작동하는 제품을 출시하는 것이 목표 — 완벽한 제품이 아님.  
> Zalo 로그인, 갤러리 그리드, 갤러리 10장 초과는 v2로 미룬다.

---

## 주차별 계획

| 주차 | 목표 | 핵심 산출물 | 위험 요소 |
|---|---|---|---|
| **1주** | 환경 세팅 + 인증 | Supabase 프로젝트 생성, Twilio Verify 연결, Google·Facebook OAuth, 전화번호 OTP 로그인 | Twilio 계정 생성·설정에 1~2일 소요 가능 |
| **2주** | 에디터 기본 폼 | 신랑신부·부모님 이름, 예식 일시, 예식 장소 (Google Maps), 인사말 | Google Maps Places API 연동 난이도 중간 |
| **3주** | 템플릿·갤러리·미리보기 | 커버 템플릿 3종 렌더러, 사진 업로드 (최대 10장), 실시간 미리보기 | 템플릿 렌더러가 가장 복잡한 컴포넌트 |
| **4주** | 계좌·공유·공개 열람 | 계좌번호 입력·표시, 공유 URL 생성 (8자리), OG 태그, 공개 청첩장 페이지 | — |
| **5주** | RSVP·방명록·마이페이지 | RSVP 폼·수집, 방명록 작성·열람, 내 청첩장 목록·복사·삭제 | — |
| **6주** | 배포·QA | Vercel 배포, 모바일 QA, 오류 수정 | 베트남 네트워크 환경 테스트 필요 |

## 종이 청첩장 통합 — Phase 2/3/4 도입

> 상세 명세: [03-설계/화면명세/07-종이청첩장.md](../03-설계/화면명세/07-종이청첩장.md)
> 상세 범위: [01-기획/MVP-범위.md](MVP-범위.md) "종이 청첩장 통합" 섹션

### Phase 2 — DIY PDF 다운로드 (출시 6~9개월 후)

| 마일스톤 | 산출물 | 위험 요소 |
|---|---|---|
| 자동 사이즈 변환 엔진 | A5/A6/바이폴드/트라이폴드 4종 | 모바일 → 종이 콘텐츠 매핑 난이도 |
| 인쇄용 PDF 생성 | Puppeteer + Sharp + ICC ColorProfile | CMYK 변환 품질 보증 |
| 표지·속지 선택 UI | 사용자 콘텐츠 선택 | UX 단순화 필요 |
| PDF 다운로드 유료화 | 100~300K VND | 결제 PG 미도입 시 무료 베타 |

→ **운영 부담 거의 없음, 시장 수요 검증 단계**

### Phase 3 — 인쇄소 파트너십 + 표준 옵션 (출시 12~18개월 후)

| 마일스톤 | 산출물 | 위험 요소 |
|---|---|---|
| 인쇄소 파트너십 | 호치민·하노이 1곳씩 | 협상 시간 1~3개월 |
| 인쇄소 발주 API 또는 수동 연동 | 자동화 우선, 수동 백업 | 인쇄소 IT 역량 차이 |
| 결제 PG 통합 | ZaloPay/MoMo/VietQR/카드 | 베트남 법인 전환 시점과 동기화 |
| 종이 재질·수량·봉투 옵션 | 매트/글로스/펄/특수지 | 인쇄소 재고 일치 필요 |
| 베트남 전국 배송 | GHN/Viettel Post/J&T API | 지방 배송 지연 가능 |
| 배송 추적 + Zalo 알림 | 실시간 상태 표시 | |

→ **표준 옵션, 사용자 80%, 1.5M~2M VND/주문**

### Phase 4 — 프리미엄 옵션 (출시 18~24개월 후)

| 마일스톤 | 산출물 | 위험 요소 |
|---|---|---|
| 컬러 감리 워크플로우 | 인쇄소 작업자 검수 SLA 24~48시간 | 인쇄소 인력 부담 |
| 테스트 인쇄 + 고객 배송 | 1~2장 시험 인쇄 → 고객 배송 → 컨펌 | 시간 +3~7일, 비용 +250~650K |
| 디자이너 검수 (옵션) | 자동 변환 후 디자이너 1차 검수 | 디자이너 채용·외주 |
| 답례품 라벨·좌석 배치표 | mail merge 자동 | 데이터 입력 UX 단순화 |

→ **프리미엄 옵션, 사용자 20%, 2.5M~3.5M VND/주문**

## 종이 청첩장 — 페이지 유형 3종 매트릭스

> 사용자가 페이지 수 먼저 선택 → 해당 유형 고정 기능만 노출.
> 상세: [03-설계/화면명세/07-종이청첩장.md](../03-설계/화면명세/07-종이청첩장.md) 섹션 1-A

| 유형 | 페이지 | 사이즈 | 등급 | 가격 (50장 기준) |
|---|---|---|---|---|
| **①** | 2페이지 | A6 / A5 (1장 양면) | 미니멀 | ~1.0M VND |
| **②** ⭐ | 4페이지 | A5 2장 / 바이폴드 | 표준 | 1.5M~2M VND |
| **③** | 6+페이지 | A5 3장+ / 트라이폴드 / 소책자 | 프리미엄 | 2.5M~3.5M VND |

각 유형별 핵심 기능:
- **유형 ①**: 이름·사진·날짜 + 메인 청첩장 QR + VietQR(선택)
- **유형 ②**: + 인사말·일시·장소 상세 + 양가 부모 + 메인 QR + VietQR + RSVP QR(선택)
- **유형 ③**: 모든 기능 + 갤러리 + QR 7종 (메인·RSVP·방명록·갤러리·음악·지도·VietQR)

## Concierge MVP 운영 모델 (Phase 2)

상세: [02-정책/운영-정책.md](../02-정책/운영-정책.md) 섹션 8

| 단계 | 처리 시간 | 1인 처리량 | 자동화율 |
|---|---|---|---|
| Phase 2 (출시 6~9개월) | 3~5일 | 월 100~150건 | 0% (수동) |
| Phase 3 (12~18개월) | 1~2일 | 월 500~2,000건 | 80% |
| Phase 4 (18~24개월) | 즉시~1일 | 월 2,000건+ | 99% |

→ 첫 100~500건은 **사람이 직접 검수·발주** = 시장 검증 + 인쇄소 신뢰 + 시행착오 학습 데이터 축적.
