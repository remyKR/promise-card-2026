## 1. 청첩장 유효 기간

| 구분 | 공개 URL 유효 기간 | 만료 후 처리 | 데이터 삭제 |
|---|---|---|---|
| **MVP (무료)** | 예식일 + 30일 | URL 비활성화, 소유자 접근 유지 | 비활성화 후 90일 |
| **v2 기본 결제** | 예식일 + **1년** | URL 비활성화, 소유자 접근 유지 | 비활성화 후 90일 |
| **v2 연장** | 연 소액 결제로 1년씩 연장 | — | — |

> **"URL 비활성화 ≠ 데이터 삭제"**  
> 만료되면 하객용 공유 URL(`/card/[code]`)만 차단됨. 소유자는 로그인 후 `/my/cards`에서 계속 접근·다운로드 가능. 데이터는 90일 유예 후 삭제.

> **Draft 카드** (미결제·미공개): 생성 후 **30일** 내 공개 전환 없으면 자동 삭제.

## 3. 업로드 제한

| 항목 | MVP 제한 | v2 제한 |
|---|---|---|
| 갤러리 사진 수 | 최대 **10장** | 최대 20장 |
| 입력 파일 형식 | JPG, JPEG, PNG, WebP, HEIC | 동일 |
| 입력 파일 크기 | 장당 최대 **20MB** (변환 전) | 동일 |

| 청첩장 수 | 제한 없음 | 동일 |

## 3-1. 사진 처리 규격 (서버 사이드 자동 처리)

업로드된 원본 파일은 서버에서 자동 변환 후 저장된다. 원본은 보존하지 않는다.

| 용도 | 최대 해상도 | 포맷 | 품질 | 예상 용량 |
|---|---|---|---|---|
| **갤러리 원본** (저장·열람용) | 장변 1920px | WebP | 85% | 300~600KB |
| **갤러리 썸네일** (에디터 미리보기용) | 장변 400px | WebP | 75% | 20~50KB |
| **약도 이미지** | 장변 1200px | WebP | 80% | 100~300KB |

**처리 파이프라인**

```
클라이언트: 파일 선택 (JPG / PNG / WebP / HEIC, 최대 20MB)
    ↓
Next.js API Route: 형식·용량 유효성 검사
    ↓
sharp 라이브러리: HEIC 디코딩 → WebP 변환 → 리사이즈
    ↓
Supabase Storage (private 버킷): 갤러리 원본 + 썸네일 저장
    ↓
클라이언트: Signed URL로 표시 (1시간 만료)
```

**절감 효과**: 원본 평균 5~10MB → 변환 후 평균 450KB → **카드 1개 총 스토리지 약 4.5MB**

## 4. 금지 콘텐츠

다음 콘텐츠는 업로드 및 입력이 금지됩니다:

- 타인의 초상권·저작권을 침해하는 사진
- 혐오·폭력·음란 콘텐츠
- 허위 정보를 통한 사기 목적 사용
- 스팸·광고성 청첩장 대량 생성

위반 시 사전 경고 없이 계정 정지 및 데이터 삭제 가능.

## 8. 종이 청첩장 정책 (v2~v3, 3단계 도입)

> 상세 명세: [03-설계/화면명세/07-종이청첩장.md](../03-설계/화면명세/07-종이청첩장.md)

### 8-1. 도입 단계

| Phase | 시점 | 범위 |
|---|---|---|
| **Phase 2** | 출시 6~9개월 후 | DIY PDF 다운로드 (셀프 인쇄) |
| **Phase 3** | 출시 12~18개월 후 | 인쇄소 파트너십 + 표준 옵션 (본인쇄·배송) |
| **Phase 4** | 출시 18~24개월 후 | 프리미엄 옵션 (컬러 감리·테스트 인쇄·고객 컨펌) |

### 8-2. 페이지 유형 3종 (기능 고정)

사용자가 페이지 수 먼저 선택 → 해당 유형의 고정 기능 세트만 노출.

| 유형 | 페이지 수 | 사이즈 | 등급 | 주요 기능 |
|---|---|---|---|---|
| **①** | 2페이지 | A6 / A5 (1장 양면) | 미니멀 | 이름·사진·날짜 + 일시·장소 한 줄 + 메인 청첩장 QR + VietQR(선택) |
| **②** ⭐ | 4페이지 | A5 2장 / 바이폴드 | 표준 | + 인사말 + 일시·장소 상세 + 양가 부모 + 감사 + 메인 QR + VietQR + RSVP QR(선택) |
| **③** | 6+페이지 | A5 3장+ / 트라이폴드 / 소책자 | 프리미엄 | 모든 기능 + 갤러리 + QR 7종 (메인·RSVP·방명록·갤러리·음악·지도·VietQR) |

### 8-3. QR 코드 통합 정책 (종이 → 모바일 진입점)

종이의 정적 한계를 모바일 청첩장 페이지로 보완하는 핵심 차별화:

| QR 종류 | 연결 페이지 | 적용 유형 |
|---|---|---|
| **메인 청첩장 QR** ⭐ 필수 | 모바일 청첩장 메인 URL | ① ② ③ |
| **VietQR (송금)** | VietQR.io 자동 생성 | 🔘 ① / ② / ✅ ③ |
| **RSVP QR** | 참석 확인 페이지 | 🔘 ② / ✅ ③ |
| **방명록 QR** | Lời chúc 작성 페이지 | ✅ ③ |
| **갤러리 QR** | 모바일 갤러리 전체 | ✅ ③ |
| **음악 QR** | 배경음악 재생 | 🔘 ③ |
| **지도 QR** | Google Maps 길안내 | 🔘 ② / ✅ ③ |

### 8-4. 가격·수량 정책 (Phase 3)

| 수량 | 단가 (장당, VND) | 총액 (50장 기준) |
|---|---|---|
| 50장 | 30~40K | 1.5M~2M |
| 100장 | 25~35K | 2.5M~3.5M |
| 200장 | 20~30K | 4M~6M |
| 500장+ | 15~25K | 7.5M+ |

베트남 시장 평균 청첩장 발송: **300~500장** (1커플 기준), **2회 결혼식 시 600~1,000장** (60~70% 커플 해당).

### 8-7. 결제·환불 정책

| 항목 | 내용 |
|---|---|
| 결제 시점 | 본인쇄 직전 100% 또는 50% 선결제 + 50% 본인쇄 직전 |

| **인쇄 미스 (인쇄소 책임)** | 재인쇄 무료 |
| **배송 분실** | 배송사 보상 + 재발송 |

### 8-9. 데이터 동기화

- 모바일 청첩장 입력 데이터 자동으로 종이에 반영

- 사용자가 종이용으로 별도 수정 가능 (선택)
