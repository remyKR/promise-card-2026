## 1. 지원 로그인 수단

| 수단 | MVP | 이유 |
|---|---|---|
| **Google OAuth** | ✅ | 글로벌 표준, Supabase 기본 지원 |

---

## 2. 인증 플로우

### 2.1 신규 가입

```
[1] Google 또는 Facebook으로 소셜 로그인
      ↓
[2] 전화번호 입력 (+84 베트남 번호 필수)
      ↓
[3] SMS OTP 발송 (6자리, 유효 시간 5분)
      ↓
[4] OTP 인증 완료
      ↓
[5] 가입 완료 → 청첩장 만들기로 이동
```

### 2.2 재로그인

```
[1] Google 또는 Facebook으로 소셜 로그인
      ↓
[2] 전화번호 인증 완료 계정 → 바로 서비스 진입
```

---

## 3. 인증 상태

| 상태 | 설명 | 접근 가능 범위 |
|---|---|---|
| `guest` | 비로그인 | 공개 청첩장 열람만 |
| `pending_phone` | 소셜 인증 완료, 전화 미인증 | 전화번호 인증 페이지만 |
| `verified` | 전화 인증 완료 | 전체 서비스 |

---

## 4. OTP 규칙

| 항목 | 값 |
|---|---|
| OTP 자리수 | 6자리 숫자 |
| 유효 시간 | 5분 |
| 재발송 대기 | 60초 |
| 최대 시도 횟수 | 5회 (초과 시 10분 잠금) |
| 전화번호 국가코드 | +84 (베트남) 기본값 |

---

## 5. Decree 147 준수 사항

Decree 147/2024/ND-CP는 베트남 내 온라인 서비스에 전화번호 인증을 의무화합니다.

- 소셜 로그인만으로는 서비스 이용 불가
- 전화번호는 베트남 번호(+84)를 원칙으로 함
- 인증 완료 전까지 콘텐츠 생성 불가

---

## 6. 세션 관리

| 항목 | 방식 |
|---|---|
| 인증 토큰 | Supabase JWT |
| Refresh Token | 자동 갱신 |
| 세션 만료 | 7일 (재로그인 불필요) |
| 로그아웃 | 클라이언트 토큰 삭제 + Supabase 세션 무효화 |

## 7. 보안 규칙

- 환경변수로만 API 키 관리 (`.env.local`, git 제외)
- Supabase RLS: 본인 데이터만 수정·삭제 가능
- HTTPS 필수 (Vercel 기본 제공)
- OTP SMS 발송은 서버사이드에서만 처리
