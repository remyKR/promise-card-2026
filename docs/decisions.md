- **결정**: 다음 토큰 추가
  - `number/56` (Primitives - Numbers)
  - `size/xl` 매핑 변경 (`number/64` → `number/56`)

- **이유**: 데어무드(theirmood) 등 모바일 청첩장 표준 기능. 베트남 시장에서도 음악 자동재생은 결혼식 분위기 연출의 핵심. 누락 시 경쟁력 ↓.
- **구현**: 기본 라이브러리(royalty-free 10~15곡) + 자동·반복 재생 + 게스트 플로팅 컨트롤. iOS Safari 자동재생 제한은 "Tap để mở nhạc" 안내 토스트로 처리.
- **v2**: 음원 직접 업로드(MP3/M4A 5MB), 페이드 효과
- **참고**: [02-에디터.md] 섹션 7

### 인사말(Lời mời) 별도 섹션으로 추가 — 섹션 3 신설
- **결정**: "인사말"을 에디터의 별도 섹션(섹션 3)으로 추가. Figma에서 이미 작업되어 있으나 명세에 누락되어 있던 것을 반영.

- **구성**: 제목(선택) + 본문(필수, 리치 텍스트) + 기본 샘플 템플릿 2~3종 + 정렬 옵션
- **영향**: 02-에디터.md 섹션 번호 재정렬 (예식 일시 3→4, 예식 장소 4→5, 갤러리 5→6, 축의금 6→7, 배경음악 7→8, RSVP 8→9, 방명록 9→10, 추가 콘텐츠 10→11, 공유 11→12, 순서 변경 12→13)

### 베트남어 UI 용어 표준화
- **결정**: 다음 표기를 표준으로 채택
  - 선택(필드) → `Tùy chọn` (필수: `Bắt buộc`)
  - 은행 → `Ngân hàng`
  - 계좌 (그룹·라벨) → `Tài khoản`
  - 계좌 이름 (필드 라벨) → `Tên tài khoản`
  - 예금주 → `Tên chủ tài khoản`
  - 계좌번호 → `Số tài khoản`
  - 신랑측/신부측 (자유 이름 예시) → `Nhà trai` / `Nhà gái`
  - 수정·편집 → `Chỉnh sửa` (또는 짧게 `Sửa`)
  - 완료 → `Xong` / `Hoàn tất`
  - 추가 → `Thêm` / `+ Thêm tài khoản`
  - 표시 옵션 → `Tùy chọn hiển thị`
- **이유**: UI 일관성·번역 통일. 향후 다른 화면 작업 시 동일 용어 사용.
