## 2026-04-23

### 도구 선택
- **결정**: Claude Design 대신 **Claude Code**로 개발 진행
- **이유**: 직접 Figma 디자인 가능 + 풀스택(프론트+백엔드) 모두 필요하기 때문

### 개발 방식
- **결정**: **바이브 코딩** 방식 — Figma 디자인 후 Claude Code로 구현
- **팀 구성**: 본인(디자인·기획) + 원희님(프론트) + Claude(구현 보조)

### 레포지토리 전환
- **결정**: 구 레포(`remyKR/promisecard`)를 현재 레포(`remyKR/promise-card-2026`)로 마이그레이션
- **이유**: 기존 기획 문서는 활용하되 디자인 시스템 파일은 초기화하고 재시작

### 기술 스택 확정
- **결정**: Next.js 15 + React 19 + Tailwind CSS 4 + Supabase + pnpm
- **제외 확정**: `next-intl` (다국어 불필요, 베트남어 단일 언어로 진행)
- **이유**: 단일 언어(베트남어)로 충분, 복잡도 줄이기

### 비즈니스 방향
- **결정**: **베트남 법인** 설립 예정 (거주 비자 목적 포함)
- **결정**: 초기 MVP는 **무료 서비스**로 출시 → 법인 설립 후 결제 연동(Phase 2)
- **타겟 시장**: 베트남 거주 커플 (한국인 + 베트남인)

### 레퍼런스
- **결정**: 데어무드(`theirmood.com`) 참고하되 일부 고난이도 기능 제외
- **제외 기능(MVP)**: Zalo 로그인, 결제 연동, 배경음악, 오프닝 애니메이션, 하객 사진 QR

### 개발 일정 (예상)
- **결정**: MVP 기준 6주 목표
  - 1주차: 환경 세팅, Supabase 연결, 로그인
  - 2주차: 청첩장 에디터 폼
  - 3주차: 템플릿 3종, 실시간 미리보기
  - 4주차: 갤러리, 지도, 공유 URL
  - 5주차: RSVP, 방명록, 마이페이지
  - 6주차: Vercel 배포, 버그 수정

### 타겟 사용자 명확화
- **결정**: 서비스 핵심 타겟은 **베트남인 커플 (베트남인 × 베트남인)**
- **이유**: 초기 문서에 "베트남 거주 한국인 커플 1순위"로 기재되어 있었으나, 실제 시장 볼륨은 베트남인 커플이 훨씬 큼
- **제외**: 한국인 × 베트남인 혼합 커플은 타겟 아님

### 청첩장 복사하기 기능 추가 (P1)
- **결정**: "청첩장 복사하기" 기능을 MVP P1으로 추가
- **이유**: 베트남 결혼 문화상 60~70%의 커플이 신부 고향·신랑 고향 각각 결혼식을 2회 진행. 두 청첩장은 장소·날짜만 다르고 나머지(사진·테마·계좌번호)는 동일 → 복사 기능 없으면 처음부터 다시 만들어야 하는 불편함 발생
- **비즈니스 임팩트**: 건당 결제 모델 기준 커플당 평균 1.6~2배 매출 발생

## 2026-04-23 (2차 세션)

### 경쟁사 분석 완료
- **결정**: 베트남 모바일 청첩장 5개 플랫폼(CineLove, Chung Đôi, Vesey, meWedding, iWedding) 가격·기능 분석 완료
- **이유**: 가격 정책 및 무료/유료 구조 결정에 앞서 시장 현황 파악 필요
- **인사이트**:
  - 무료 플랜 + 건당 유료 전환이 베트남 시장 표준 모델 (Chung Đôi: 199,000 VND / Vesey: 129,000 VND)
  - 구독제(월정액)는 없음 — 모두 건당 일회성 결제 또는 1~5년 라이선스
  - 결제 수단: 모두 베트남 은행 계좌이체 + QR코드 (카드/해외결제 없음)
  - 결제 시점: 공개(공유) 직전 또는 3일 무료 체험 후 결제 (사전 결제 없음)
- **가격 전략**: 고가 전략 (경쟁사 대비 프리미엄 포지셔닝) — 구체적 가격은 추후 책정

### 종이 청첩장 서비스 v2 추가
- **결정**: 디지털 청첩장 → 종이 청첩장 자동 변환·인쇄·배송 서비스를 v2 기능으로 추가
- **이유**: 베트남은 어르신·친척에게 종이 청첩장 전달 문화가 여전히 강함. 도시 커플 40~60%가 사진 포함 종이 청첩장 사용 → 디지털 데이터 재활용으로 경쟁사 대비 간편한 경험 제공 가능
- **운영 방식**: 직접 운영 아닌 파트너 인쇄소 연결 모델 (중개 수수료)
- **템플릿**: 사진형·전통형 2종
- **전제 조건**: 베트남 법인 설립 + 인쇄소 파트너 계약

### 구글 드라이브 사진 가져오기 v2 추가
- **결정**: 갤러리 업로드에 "구글 드라이브에서 가져오기" 기능 v2 추가
- **이유**: 베트남 웨딩 스튜디오가 편집본 수십~수백 장을 Google Drive 링크로 전달하는 게 표준. 커플이 폰에 저장 후 다시 업로드하는 번거로움을 제거
- **구현**: Google Drive Picker API (OAuth 연동)

### Zalo 로그인 MVP 포함으로 변경
- **결정**: Zalo 로그인을 P3(v2) → **P1(MVP)** 으로 변경
- **이유**: 베트남 핵심 타겟(베트남인 커플) 대상 서비스에서 Zalo는 필수. 한국이 아닌 베트남 현지에서 직접 테스트 예정
- **변경 파일**: MVP-범위.md, 아키텍처.md, 인증-정책.md
