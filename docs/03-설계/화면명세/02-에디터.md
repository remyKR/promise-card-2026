## 화면 구조

```
┌─────────────────────────────────────────┐
│  [미리보기 영역]   │  [설정 패널]         │
│  (좌측 or 상단)   │  (우측 or 하단)      │

## 섹션 1. 메인 화면 편집

> 청첩장의 첫인상(메인 화면)을 결정하는 영역. 헤더 배지: [`Bắt buộc`] (필수)
> Figma 노드: `1:1300`

### 1-1. 테마 선택 (Chọn giao diện)
- **작동**: 사전 큐레이션된 테마 카드 그리드에서 선택
- **시각**: 첫 카드 선택 시 우상단 체크 표시
- **전체 보기**: 헤더 우측 [`Xem tất cả`] 버튼 → 풀스크린 모달
- **모달 프리셋**: 약 8종 내외 ([06-템플릿.md](06-템플릿.md) 참조)
- **모달 인터랙션**:
  - 카드 탭 → 즉시 메인 미리보기 적용 (모달 유지)
  - [`Xong`] → 확정 / [`Hủy`] → 롤백

### 1-2. 결혼식 사진 (Ảnh cưới)
- **작동**: 메인 화면 커버 사진 업로드
- **UI**: 카메라 아이콘 박스 (탭하여 사진 선택)
- **포맷**: JPG, PNG, WebP, HEIC
- **자동 변환**: 업로드 시 WebP 변환·리사이즈

### 1-3. 글자 색상 (Màu chữ)
- **작동**: 청첩장 텍스트 색상 선택
- **옵션**: 3색 (체크 / 검정 / 회색)
- **시각**: 선택된 색상에 체크 표시

### 1-4. 배경 효과 (Hiệu ứng nền)
- **작동**: 청첩장 화면에 떨어지는 파티클 애니메이션 효과
- **옵션** (4종):
  - **Không** (없음) — 기본값 ✓
  - **Tuyết** (눈)
  - **Đom đóm** (반딧불)
  - **Hoa anh đào** (벚꽃)
- **시각**: 선택된 옵션 갈색 배경 강조 + 체크

## 섹션 2. 기본 정보 (Thông tin cơ bản)

> 신랑·신부·부모님 이름을 입력하는 영역. 헤더 배지: [`Bắt buộc`] (필수)
> Figma 노드: `1:1406`
> 화면 순서: 신부(Cô dâu) → 신랑(Chú rể) (베트남 결혼 관행 반영)

### 2-1. 신부 (Cô dâu)
- **Họ** (성): 단일 입력 필드
- **Tên** (이름): 단일 입력 필드
- **Họ tên bố** (아버지 이름): 성+이름 통합 입력
- **Họ tên mẹ** (어머니 이름): 성+이름 통합 입력

### 2-2. 신랑 (Chú rể)
- **Họ** (성)
- **Tên** (이름)
- **Họ tên bố** (아버지 이름)
- **Họ tên mẹ** (어머니 이름)

### 2-3. 안내 문구
- 베트남어: `Thông tin bố·mẹ có thể bỏ qua.`
- 의미: 부모님 정보 입력하지 않아도 됨 (자동 숨김 처리)

## 섹션 4. 예식 일시 (Thời gian tổ chức)

> Figma 노드: `97:12`
> Figma 헤더: `Vui lòng chọn` / `thời gian`

### 4-1. 날짜 선택
- **베트남어 표시**: `Ngày {DD} tháng {MM} năm {YYYY}`
- **작동**: 달력 UI에서 날짜 선택
- **표시 포맷**: "2026년 4월 22일 수요일" (자동 요일 계산)
- **최솟값**: 오늘 이후 날짜만 선택 가능

### 4-2. 시간 선택
- **베트남어 표시**: `{HH}h {MM}phút` (예: `16h 30phút`)
- **작동**: 시 + 분(5분 단위) 드롭다운
- **표시 포맷**: "오후 6시 30분" (베트남어 24시간제)

### 4-3. 표시 방식 — 캘린더 (Lịch) — 토글
- **작동**: 청첩장 내 한 달 달력 표시, 예식일 강조
- **시각**: 활성 시 갈색 배경 + ✓ 표시
- **스타일**: 미니 달력, 예식일 원형 배경

### 4-4. 표시 방식 — D-Day 카운트다운 (Đếm ngược) — 토글
- **작동**: 예식일까지 남은 일수·시간·분·초 실시간 카운트다운
- **시각**: 활성 시 갈색 배경 + ✓ 표시
- **포맷**: "DAYS 00 : HOURS 00 : MIN 00 : SEC 00"

### 4-5. D-Day 메시지 커스터마이징
- **작동**: 카운트다운 영역에 표시될 메시지 리치 텍스트 편집
- **에디터**: Bold / Italic / 정렬 3종 / 색상 4종 (검정·갈색·진회색·연회색)
- **기본 메시지** (베트남어):
  ```
  Chỉ còn (D-Day) ngày nữa là đến ngày trọng đại của (chú rể) và (cô dâu).
  ```
- **변수 자동 치환**:
  - `(D-Day)` — 남은 일수 (자동 계산)
  - `(chú rể)` — 신랑 이름
  - `(cô dâu)` — 신부 이름

## 섹션 7. 축의금 (Mừng cưới)

> 게스트가 디지털 송금으로 축의금을 전달할 수 있도록 계좌 정보·VietQR을 제공.
> 베트남 표준 결혼 문화(신랑/신부 가문 단독 주최) 반영하여 그룹 분리 없이 평면 계좌 리스트로 운영.
> Figma 노드: `130:2`
>
> **Figma 헤더 (2줄)**:
> ```
> Vui lòng cài đặt
> mục mừng cưới
> ```

### 7-1. 섹션 표시 여부 (Master Toggle)
- **베트남어 라벨**: `Hiển thị mục mừng cưới`
- **작동**: 토글 ON/OFF로 게스트 화면에서 섹션 전체 노출 제어
- **OFF 시**: 게스트 화면에서 섹션 통째 사라짐 (데이터 보존)
- **헤더 배지**: [`Tùy chọn`] (선택 섹션)

### 7-2. 안내 메시지 (Lời nhắn về tài khoản)
- **작동**: 계좌 영역 상단에 표시할 안내 메시지 (리치 텍스트)
- **에디터**: Bold / Italic / 정렬 / 색상 4종 (black / brand / gray / light gray)
- **빈 값 허용**: 미작성 시 영역 미노출

### 7-3. 계좌 리스트
- **작동**: 평면 계좌 리스트, 자동 인덱스(`Tài khoản N`)
- **그룹 분리 X**: 자유 이름 부여 (예: `Nhà trai`, `Nhà gái`)

#### 7-3-1. 계좌 카드 필드
- **인덱스 라벨**: `Tài khoản 1`, `Tài khoản 2` (자동, 편집 불가)
- **계좌 이름** (선택, `Tên tài khoản`): 자유 텍스트
- **은행** (`Ngân hàng`): 드롭다운 (VietQR.io 50+ 베트남 은행 + 로고)
- **예금주명** (`Tên chủ tài khoản`)
- **계좌번호** (`Số tài khoản`): 4자리 단위 공백 자동 (예: `0011 0023 4567`)

#### 7-3-2. 카드 액션
- **순서 조정 (▲▼)**: 카드 단위 위·아래 이동
- **편집 토글 (`Sửa` ↔ `Xong`)**:
  - 보기 모드: 평문, 밑줄 없음
  - 편집 모드: 인풋 활성, 밑줄, placeholder

#### 7-3-3. 계좌 추가
- **버튼**: `+ Thêm tài khoản` (다크 배경, 흰 글씨, 리스트 하단 중앙)
- **동작**: 빈 카드 생성 → 자동 편집 모드 → 인덱스 자동 부여

#### 7-3-4. 계좌 삭제
- **노출**: 편집 모드 (`Xong`) 카드 내부에서만
- **확인**: 삭제 시 컨펌 다이얼로그 1회

### 7-4. VietQR 자동 생성
- **베트남어 라벨**: `Hiển thị mã VietQR`
- **핵심**: 사용자가 QR 이미지 업로드 X. 입력 정보로 자동 생성
- **생성 방식**: 은행 BIN + 계좌번호 + 예금주명 → VietQR.io API (`img.vietqr.io`)
- **표시 토글**: 섹션 하단 토글로 ON/OFF
- **표시 위치 (게스트)**: 청첩장 열람 화면 각 계좌 카드 내부
- **에디터 미리보기**: 작성자 본인 QR 미리 확인 가능
- **조건**: 은행/예금주/계좌번호 3개 모두 입력 시 QR 생성. 1개라도 비면 안내
- **미지원 은행**: "Ngân hàng khác" 선택 시 QR 자동 비활성화

### 7-5. 게스트 인터랙션 (열람 화면 연동)

## 섹션 8. 배경음악 (Nhạc nền)

> 청첩장 열람 시 배경음악 자동 재생.
> Figma 노드: `165:278`

### 8-1. 섹션 표시 여부 (Master Toggle)
- **베트남어 라벨**: `Sử dụng nhạc nền` (배경음악 사용하기)
- **작동**: 토글 ON/OFF
- **위치**: 헤더 바로 아래, brand-subtle 배경 강조 박스

### 8-2. 음원 선택 — 기본 라이브러리
- **곡 수**: 약 10~15곡 (royalty-free, 베트남 결혼식 인기곡 + 인스트)
- **카테고리**: Lãng mạn / Truyền thống / Hiện đại / Không lời
- **단일 선택**: 한 번에 1곡 (라디오 방식)

#### 8-2-1. 트랙 카드 구조
- **선택 인디케이터** (24×24): 라디오 버튼 (선택 시 체크 + 브랜드 컬러)
- **앨범 썸네일** (80×56): 음원 표지 또는 카테고리 아이콘
- **트랙 정보** (Table Cell):
  - 윗줄 (작은 회색): 음악 설명
  - 아랫줄 (굵은 본문): 음악 이름
- **재생 컨트롤** (36×36 원형):
  - 미재생: ▶ Play
  - 재생 중: ⏸ Pause

#### 8-2-2. 트랙 선택 (단일)
- **인터랙션**: 트랙 카드 어디든 탭 → 선택
- **시각 강조**:
  - 라디오 버튼 체크
  - 트랙 제목 하단 underline
- **즉시 반영**: 청첩장에 해당 곡 적용

#### 8-2-3. 미리듣기·재생 (인라인)
- **Play 탭**: 즉시 재생
- **재생 중 Pause 탭**: 일시정지
- **다른 트랙 Play 탭**: 이전 곡 정지 후 새 곡 재생
- **시간**: 풀버전 재생 (30초 제한 X)

### 8-3. 재생 옵션
- **자동 재생** (`Tự động phát`): 기본 ON, Pill 토글
  - 위치: 트랙 리스트 하단, divider 아래
- **반복 재생** (`Lặp lại`): **항상 적용** (토글 없음, 자동)
  - 끄려면: 마스터 토글 OFF

### 8-4. 게스트 화면 표시 (Viewer)
- **위치**: 우측 하단 플로팅 버튼 (스크롤 따라 고정, z-index 최상위)
- **상태별 표시**:
  - 재생 중: 음표 아이콘 + 애니메이션 파동
  - 일시정지: 음소거 아이콘 (정적)
- **인터랙션**: 탭 → 재생/일시정지 토글

### 8-5. iOS Safari 자동재생 제한 처리
- **이유**: iOS 11+ Safari 자동재생 차단
- **방안**:
  1. 페이지 로드 시 `<audio>` 객체 생성 (재생 미시작)
  2. 첫 진입 시 안내 토스트 "Tap để mở nhạc"
  3. 사용자 첫 탭 → 음악 재생 시작
  4. 이후 플로팅 버튼으로 자유 컨트롤

### 8-6. 베트남 환경 최적화
- **비트레이트**: 96~128kbps
- **곡 길이**: 평균 2~3분 (반복 재생 전제)
- **CDN**: Supabase Storage + Cloudflare 자동 캐시
- **pre-load**: `metadata only`
