# 청첩장 제작하기 — 화면 기능 명세서

> 경로: `/create`  
> 인증: 로그인 필요  
> 레퍼런스: 데어무드 모바일청첩장 제작하기  
> 버전: v1.0 | 작성일: 2026-04-23

---

## 화면 구조

```
┌─────────────────────────────────────────┐
│  [미리보기 영역]   │  [설정 패널]         │
│  (좌측 or 상단)   │  (우측 or 하단)      │
│                   │  섹션 탭 목록        │
│  실시간 렌더링     │  각 섹션 설정 폼     │
└─────────────────────────────────────────┘
```

- **모바일**: 상단 미리보기(50%) + 하단 스크롤 설정 패널
- **데스크톱**: 좌측 미리보기(고정) + 우측 설정 패널(스크롤)
- 설정값 변경 시 미리보기 **실시간 반영**

## 섹션 1. 테마·디자인

### 1-1. 글꼴 선택
- **작동**: 글꼴 목록에서 하나를 탭하면 청첩장 전체 텍스트에 즉시 적용됨
- **옵션**: 고운돋움 / 리디바탕 / 이롭게바탕 / 둥근바람 / 연성체 / 노토산스
- **크기**: 보통 / 크게 / 더 크게 — 기본값은 "보통"
- **미리보기**: 글꼴 목록 각 항목에 해당 폰트로 렌더링된 샘플 텍스트 표시
- **MVP**: ✅ 포함 | **우선순위**: P2

### 1-2. 배경 색상
- **작동**: 색상 피커(또는 프리셋 팔레트)에서 색상 선택 → 청첩장 전체 배경색 변경
- **기본값**: 흰색(#FFFFFF)
- **제한**: 가독성 보장을 위해 너무 어두운 색 선택 시 텍스트 색상 자동 반전 고려
- **MVP**: ✅ 포함 | **우선순위**: P2

### 1-3. 강조 색상
- **작동**: 소제목, 구분선, 버튼 등 포인트 요소에 적용되는 색상 선택
- **연동**: 배경 색상과 자동 대비 계산하여 가독성 유지
- **MVP**: ✅ 포함 | **우선순위**: P2

### 1-4. 배경 패턴
- **작동**: 청첩장 배경 전체에 반복 패턴 이미지를 오버레이로 적용
- **옵션**: 없음 / 작은 꽃 / 큰 꽃
- **투명도**: 패턴은 반투명(약 10~20% opacity)으로 텍스트 가독성 유지
- **MVP**: ❌ 제외(v2) | **우선순위**: P3

### 1-5. 배경 이펙트
- **작동**: 청첩장 화면 위에 떨어지는 파티클 애니메이션 지속 재생
- **옵션**: 없음 / 벚꽃 / 눈 / 낙엽 / 개나리 / 안개꽃
- **성능**: CSS 애니메이션 또는 Canvas 기반 — 저사양 기기 고려 필요
- **MVP**: ❌ 제외(v2) | **우선순위**: P3

### 1-6. 등장 효과
- **작동**: 게스트가 청첩장을 스크롤할 때 각 섹션이 부드럽게 등장하는 애니메이션
- **옵션**: 없음 / 샤르륵(fade-in + slide-up)
- **구현**: Intersection Observer API 활용
- **MVP**: ❌ 제외(v2) | **우선순위**: P3

### 1-7. 확대 방지
- **작동**: 게스트 화면에서 핀치 줌 및 더블탭 확대를 비활성화
- **구현**: `<meta name="viewport" content="user-scalable=no">` 동적 삽입
- **목적**: 청첩장 레이아웃 깨짐 방지, 사진 저작권 보호
- **MVP**: ✅ 포함 | **우선순위**: P2

### 1-8. 청첩장 잠금
- **작동**: 공개 URL 접근 제어 설정
- **옵션**:
  - 공개: 누구나 접근 가능 (기본값)
  - 예식 후 잠금: 설정한 예식일 +1일 00:00부터 자동으로 비공개 전환
  - 바로 잠금: 즉시 비공개 (링크 접근 시 "청첩장이 종료되었습니다" 안내)
- **MVP**: ✅ 포함 | **우선순위**: P2

## 섹션 2. 기본 정보

### 2-1. 신랑·신부 이름 입력
- **작동**: 각각 성(họ)과 이름(tên) 분리 입력 또는 전체 이름 입력
- **표시**: 청첩장 메인 화면과 상단 헤더에 "신랑이름 & 신부이름" 형태로 표시
- **유효성 검사**: 빈값 저장 방지, 최대 20자 제한
- **MVP**: ✅ 포함 | **우선순위**: P1

### 2-2. 부모님 이름 입력
- **작동**: 신랑/신부 각각 아버지·어머니 이름 입력 (선택 사항)
- **표시**: 청첩장 하단 혼주 안내 영역에 "신랑 아버지 ○○○ · 어머니 ○○○" 형태
- **생략 처리**: 미입력 시 해당 라인 자동 숨김
- **MVP**: ✅ 포함 | **우선순위**: P1

### 2-3. 故人 표기
- **작동**: 故人 체크박스 선택 시 해당 이름 앞에 "故" 텍스트 또는 국화 아이콘 표시
- **옵션**: 텍스트(故) 표기 / 국화 이미지 표기
- **MVP**: ✅ 포함 | **우선순위**: P2

### 2-4. 세례명 입력
- **작동**: 신랑·신부·부모님 각각 세례명 별도 입력 가능
- **표시**: 이름 옆에 소괄호로 "이름 (세례명)" 형태 표시
- **생략 처리**: 미입력 시 자동 숨김
- **MVP**: ❌ 제외(v2) | **우선순위**: P3

### 2-5. 신부측 먼저 표시
- **작동**: 토글 ON 시 청첩장 전체에서 신부 정보가 신랑 정보보다 앞에 표시
- **적용 범위**: 혼주 안내, 이름 표기 등 (※ 축의금 계좌 순서는 별개 — 섹션 6의 ▲▼ 수동 정렬로 관리)
- **MVP**: ✅ 포함 | **우선순위**: P2

## 섹션 3. 예식 일시

### 3-1. 날짜 선택
- **작동**: 달력 UI에서 날짜 탭 → 선택된 날짜가 청첩장에 자동 포맷으로 표시
- **표시 포맷**: "2026년 4월 22일 수요일" (자동 요일 계산)
- **최솟값**: 오늘 이후 날짜만 선택 가능
- **MVP**: ✅ 포함 | **우선순위**: P1

### 3-2. 시간 선택
- **작동**: 시(오전7시~오후11시) + 분(00~55분, 5분 단위) 드롭다운으로 선택
- **표시 포맷**: "오후 6시 30분" (베트남어: "18 giờ 30 phút")
- **MVP**: ✅ 포함 | **우선순위**: P1

### 3-3. 표시 방식 — 캘린더
- **작동**: 청첩장 내 한 달 달력이 표시되며 예식일에 표시
- **스타일**: 미니 달력, 예식일 강조 표시(원형 배경)
- **MVP**: ✅ 포함 | **우선순위**: P2

### 3-4. 표시 방식 — D-Day 카운트다운
- **작동**: 예식일까지 남은 일수·시간·분·초를 실시간으로 카운트다운
- **포맷**: "DAYS 00 : HOURS 00 : MIN 00 : SEC 00"
- **예식 당일**: "○○의 결혼식이 오늘입니다!" 문구로 전환
- **예식 후**: 카운트다운 영역 숨김 또는 "결혼을 축하합니다" 문구 표시
- **MVP**: ✅ 포함 | **우선순위**: P2

### 3-5. D-Day 문구 커스텀
- **작동**: "○○와 ○○의 결혼식이 D-Day 일 남았습니다." 문구에서 앞부분 직접 입력
- **변수**: `{신랑이름}`, `{신부이름}`, `{D-Day}` 자동 치환
- **MVP**: ✅ 포함 | **우선순위**: P2

## 섹션 4. 예식 장소

### 4-1. 주소 검색
- **작동**: 검색창에 예식장명 또는 주소 입력 → Google Maps API로 자동완성 목록 표시 → 선택 시 위도·경도·주소 자동 입력
- **국내/해외 전환**: 베트남 기준 국내(VN) 우선, 해외 직접 입력도 지원
- **MVP**: ✅ 포함 | **우선순위**: P1

### 4-2. 예식장 상세 정보 입력
- **항목**: 예식장명 / 층과 홀 이름 / 대표 연락처
- **표시**: 청첩장 오시는 길 섹션 상단에 텍스트로 표시
- **MVP**: ✅ 포함 | **우선순위**: P1

### 4-3. 지도 표시
- **작동**: 토글 ON 시 청첩장 내 Google Maps 지도 삽입
- **기능**: 핀 마커 표시, 확대/축소 조작 (지도 잠금 시 스크롤만 가능)
- **지도 높이**: 기본(250px) / 축소(150px), 줌 레벨 15~19 선택
- **MVP**: ✅ 포함 | **우선순위**: P1

### 4-4. 지도 잠금
- **작동**: 토글 ON 시 지도 드래그·핀치·클릭 비활성화
- **목적**: 게스트가 실수로 지도를 조작해 페이지 스크롤이 방해받는 것 방지
- **MVP**: ✅ 포함 | **우선순위**: P2

### 4-5. 내비게이션 앱 바로가기
- **작동**: 버튼 탭 시 기기에 설치된 지도 앱(Google Maps / Apple Maps / Zalo Maps)으로 예식장 좌표 전달하여 길 안내 시작
- **베트남**: Google Maps 딥링크 우선
- **MVP**: ✅ 포함 | **우선순위**: P1

### 4-6. 교통 수단 안내
- **작동**: 최대 4개 교통 수단 정보를 자유 텍스트로 입력
- **표시**: 아코디언 또는 탭 형태로 게스트 화면에 표시
- **예시**: "버스 — 105번, 205번 탑승 후 ○○역 하차"
- **MVP**: ✅ 포함 | **우선순위**: P2

### 4-7. 약도 사진
- **작동**: 직접 제작한 약도 이미지 파일 업로드 → 지도 아래에 표시
- **지원 형식**: JPG, PNG, WebP
- **MVP**: ✅ 포함 | **우선순위**: P2

## 섹션 5. 갤러리

### 5-1. 사진 업로드
- **작동**: 파일 선택 또는 드래그&드롭으로 최대 10장 업로드 (v2에서 20장)
- **입력 형식**: JPG, JPEG, PNG, WebP, HEIC (iPhone 기본 포맷 대응)
- **입력 크기 제한**: 장당 최대 20MB
- **서버 자동 처리**:
  - HEIC → WebP 변환 (sharp 라이브러리)
  - 장변 1920px로 리사이즈, 비율 유지
  - WebP 품질 85% 저장
  - 에디터 썸네일용 400px 버전 별도 생성
  - 변환 후 평균 300~600KB (원본 대비 ~95% 절감)
- **저장**: Supabase Storage private 버킷 → Signed URL로 제공 (1시간 만료)
- **순서 변경**: 업로드 후 썸네일 드래그로 순서 재정렬
- **상세 규격**: `02-정책/서비스-정책.md` 섹션 3-1 참조
- **MVP**: ✅ 포함 | **우선순위**: P1

### 5-1-1. 배경 제거
- **작동**: 업로드한 사진 썸네일에서 [배경 제거] 버튼 탭 → AI가 인물·피사체 외 배경을 자동 제거 → 투명 배경(PNG) 결과물로 교체
- **진입점**: 갤러리 사진 편집 모드 (썸네일 길게 탭 또는 편집 아이콘)
- **결과물**: 원본은 보존, 배경 제거된 PNG를 새 버전으로 저장
- **취소**: 원본으로 되돌리기 버튼 제공
- **구현**: remove.bg API 또는 Cloudflare AI Background Removal 연동
- **MVP**: ✅ 포함 | **우선순위**: P2

### 5-2. 갤러리 타입
- **작동**: 게스트 화면에서 사진을 표시하는 레이아웃 방식 선택
- **옵션**: (썸네일+스와이프로 통일)
  - 스와이프: 한 장씩 좌우 슬라이드
  - 썸네일: 3열 그리드 썸네일 → 탭 시 크게 보기
- **MVP**: ✅ 스와이프만 제공 | **우선순위**: P1 (그리드는 P2)

### 5-3. 팝업 뷰어
- **작동**: 사진 탭 시 전체 화면 팝업으로 표시
- **제스처**: 좌우 스와이프로 이전/다음 이동, 위아래 스와이프로 닫기
- **MVP 제외 이유**: 스와이프 갤러리는 이미 풀 너비 표시 → 팝업이 불필요. 그리드 타입 추가 시(P2) 함께 구현.
- **MVP**: ❌ 제외 | **우선순위**: P2 (그리드 갤러리 타입과 함께)

### 5-4. 줌 기능
- **작동**: 팝업 뷰어 내 핀치 투 줌으로 사진 확대
- **MVP**: ❌ 제외 | **우선순위**: P2 (팝업 뷰어와 함께)

### 5-5. 엔딩 사진·문구
- **작동**: 갤러리 마지막에 대표 사진 1장 + 감사 문구를 엔딩 카드로 표시
- **문구**: 기본 샘플 또는 직접 입력
- **MVP**: ✅ 포함 | **우선순위**: P2

## 섹션 6. 축의금 (Mừng cưới)

> 게스트가 디지털 송금으로 축의금을 전달할 수 있도록 계좌 정보·VietQR을 제공하는 영역.
> 베트남 표준 결혼 문화(신랑/신부 가문 단독 주최) 반영하여 그룹 분리 없이 평면 계좌 리스트로 운영.
> Figma 노드: `130:2`

### 6-1. 섹션 표시 여부 (Master Toggle)
- **작동**: "Hiển thị mục mừng cưới" 토글로 청첩장 열람 화면에서 섹션 전체 노출 ON/OFF
- **OFF 시**: 게스트 화면에서 축의금 섹션 통째로 사라짐. 데이터는 보존 → 다시 켜면 복원
- **선택 섹션 표시**: 헤더 우측 [`Tùy chọn`] 배지로 선택 항목임을 명시
- **MVP**: ✅ 포함 | **우선순위**: P1

### 6-2. 안내 메시지 (Lời nhắn về tài khoản)
- **작동**: 계좌 영역 상단에 표시할 안내 메시지를 리치 텍스트로 작성
- **에디터 도구**: Bold, Italic, 정렬(좌/중앙/우), 색상 4종 (black / brand / gray / light gray)
- **기본 템플릿**: "Đối với những ai không thể đến chung vui, chúng mình xin phép để lại thông tin tài khoản. Mong mọi người thông cảm và đón nhận với tấm lòng rộng mở."
- **빈 값 허용**: 메시지 미작성 시 영역 자체 미노출
- **MVP**: ✅ 포함 | **우선순위**: P1

### 6-3. 계좌 리스트
- **작동**: 계좌를 평면 리스트로 관리. 각 계좌는 자동 인덱스(`Tài khoản N`)로 식별
- **그룹 개념 폐기**: 신랑/신부측 강제 분리 없음. 각 계좌에 자유 이름 부여 (예: "Nhà trai", "Nhà gái", "Tài khoản bố mẹ")
- **이유**: 베트남은 신랑/신부 가문이 각자 결혼식을 단독 주최하는 문화 → 청첩장 1장 = 한쪽 가문 = 그룹 분리 불필요. 합동식·다중 계좌는 자유 이름으로 유연 대응 (참고: "청첩장 복사하기" 기능과 함께 동작)
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 6-3-1. 계좌 카드 필드
- **인덱스 라벨**: `Tài khoản 1`, `Tài khoản 2` ... (자동 부여, 편집 불가)
- **계좌 이름** (선택): 자유 입력 텍스트, 빈 값 허용
- **은행** (`Ngân hàng`): 드롭다운 — VietQR.io API 기반 50+ 베트남 은행 + 로고 표시
- **예금주명** (`Tên chủ tài khoản`): 텍스트 입력
- **계좌번호** (`Số tài khoản`): 숫자 입력. 표시 시 4자리 단위 공백 자동 삽입 (예: `0011 0023 4567`)

#### 6-3-2. 카드 액션
- **순서 조정 (▲▼)**: 카드 단위 위·아래 이동 (드래그 대신 명시 버튼 — 모바일 친화)
- **편집 토글 (`Sửa` ↔ `Xong`)**:
  - 보기 모드 (`Sửa` 노출): 모든 필드 평문, 밑줄 없음
  - 편집 모드 (`Xong` 노출): 인풋 활성, 밑줄 표시, placeholder 노출
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 6-3-3. 계좌 추가
- **버튼**: "+ Thêm tài khoản" (다크 배경, 흰 글씨, 리스트 하단 중앙)
- **동작**: 빈 카드 생성 → 자동으로 편집 모드 진입 → 인덱스 자동 부여
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 6-3-4. 계좌 삭제
- **노출 위치**: 편집 모드(`Xong` 상태) 카드 내부에서만 노출 (의도치 않은 삭제 방지)
- **확인**: 삭제 시 컨펌 다이얼로그 1회
- **0개 허용**: 모든 계좌 삭제 가능 (마스터 토글 OFF와 동일 효과)
- **MVP**: ✅ 포함 | **우선순위**: P1

### 6-4. VietQR 자동 생성
- **핵심**: 사용자가 QR 이미지 업로드·등록할 필요 없음. 입력된 정보로 **자동 생성**.
- **생성 방식**: 은행 BIN 코드 + 계좌번호 + 예금주명 → VietQR.io API (`img.vietqr.io`)
- **표시 토글**: "Hiển thị mã VietQR" — 섹션 하단 토글로 ON/OFF
- **표시 위치 (게스트)**: 청첩장 열람 화면 각 계좌 카드 내부
- **에디터 미리보기**: 작성자가 본인 QR 미리 확인 가능
- **조건**: 은행/예금주/계좌번호 3개 모두 입력되어야 QR 생성. 1개라도 비면 안내 문구
- **미지원 은행**: "Ngân hàng khác" 선택 시 QR 자동 비활성화 + 안내
- **MVP**: ✅ 포함 | **우선순위**: **P1** ⚠️ *기존 P3에서 격상*
- **사유**: 베트남 5개 경쟁사 중 4곳이 QR 제공 → MVP에서 누락 시 즉시 경쟁력 상실

### 6-5. 게스트 인터랙션 (열람 화면 연동)
- **계좌번호 복사**: 각 계좌 [복사] 버튼 → 클립보드 복사 → "`Đã sao chép`" 토스트
- **VietQR 탭**: 풀스크린 모달로 확대 (게스트 은행앱으로 스캔 편의성)
- **MVP**: ✅ 포함 | **우선순위**: P1

### 6-6. 노출 타입 (게스트 열람 화면)
- **MVP**: 단일 카드 리스트 (각 계좌 = 카드, 세로 스택)
- **v2**: 아코디언·스와이프 등 옵션 추가 검토
- **MVP**: ✅ 카드 리스트만 | **우선순위**: P2

### 6-7. 간편송금 연동
- **베트남**: MoMo / ZaloPay 별도 송금 버튼
- **MVP**: ❌ 제외(v2) | **우선순위**: P3
- **사유**: VietQR이 이미 모든 베트남 뱅킹 앱에서 작동 → MoMo/ZaloPay는 보조 수단

### 6-8. 변경 이력 (이전 명세 대비)
- ❌ **폐기**: 신랑/신부측 그룹 구조 → 평면 계좌 리스트 (베트남 결혼 문화 반영)
- ❌ **폐기**: "신부측 먼저 표시" 토글 → ▲▼ 수동 정렬로 대체
- ❌ **폐기**: 노출 타입 3종(아코디언/스와이프/카드) → MVP는 단일 카드 리스트만
- ❌ **폐기**: 계좌번호 은행별 하이픈 자동 포맷 → 4자리 공백 단순 포맷으로 통일
- ✅ **추가**: 안내 메시지(Lời nhắn về tài khoản) 리치 텍스트
- ✅ **추가**: 계좌별 자유 이름 (그룹 라벨 대체)
- ✅ **추가**: 인라인 수정 모드 (`Sửa`/`Xong`)
- ✅ **추가**: 카드 순서 수동 조정 (▲▼)
- ✅ **추가**: 헤더 [`Tùy chọn`] 배지
- ⬆ **격상**: VietQR 자동 생성 P3(v2) → **P1(MVP)**

## 섹션 7. 배경음악 (Nhạc nền)

> 청첩장 열람 시 배경음악을 자동 재생하여 분위기 연출.
> 데어무드 패턴 참조 + iOS Safari 자동재생 제한 고려.
> Figma: 미작업 (재제작 대상)

### 7-1. 섹션 표시 여부 (Master Toggle)
- **작동**: "Hiển thị nhạc nền" 토글로 배경음악 기능 ON/OFF
- **OFF 시**: 게스트 화면에서 음악 플로팅 버튼 미노출
- **MVP**: ✅ 포함 | **우선순위**: P1

### 7-2. 음원 선택 — 기본 라이브러리
- **곡 수**: 약 10~15곡 (베트남 결혼식 인기곡 + 인스트루멘탈)
- **카테고리**: Lãng mạn (로맨틱) / Truyền thống (전통) / Hiện đại (모던) / Không lời (인스트)
- **미리듣기**: 곡 옆 ▶ 버튼 → 30초 프리뷰
- **저작권**: royalty-free 음원만 큐레이션 (별도 라이선스 절차 불필요)
- **MVP**: ✅ 포함 | **우선순위**: P1

### 7-3. 음원 선택 — 직접 업로드
- **포맷**: MP3, M4A
- **용량**: 최대 5MB
- **시간**: 최대 3분 (반복 재생 전제)
- **저장**: Supabase Storage
- **저작권**: 사용자 책임 고지 + 약관 동의
- **MVP**: ❌ 제외(v2) | **우선순위**: P2

### 7-4. 재생 옵션
- **자동 재생** (`Tự động phát`): 기본 ON
- **반복 재생** (`Lặp lại`): 기본 ON
- **페이드 인/아웃**: v2
- **MVP**: ✅ 포함 | **우선순위**: P1

### 7-5. 게스트 화면 표시 (Viewer 연동)
- **위치**: 우측 하단 플로팅 버튼 (스크롤해도 고정, z-index 최상위)
- **상태별 표시**:
  - 재생 중: 음표 아이콘 + 애니메이션 파동
  - 일시정지: 음소거 아이콘 (정적)
- **인터랙션**: 탭 → 재생/일시정지 토글
- **MVP**: ✅ 포함 | **우선순위**: P1

### 7-6. iOS Safari 자동재생 제한 처리 (필수)
- **이유**: iOS 11+ Safari는 무음 외 자동재생 차단 → 첫 사용자 인터랙션 후에만 재생 가능
- **방안**:
  1. 페이지 로드 시 `<audio>` 객체 생성 (재생 미시작)
  2. 첫 진입 시 안내 토스트 "Tap để mở nhạc" (한 번만, dismissible)
  3. 사용자 첫 탭(어디든) 발생 → 음악 재생 시작
  4. 이후 플로팅 버튼으로 자유 컨트롤
- **MVP**: ✅ 포함 | **우선순위**: P1
- **참고**: 모든 모바일 청첩장 서비스의 표준 패턴

### 7-7. 베트남 환경 최적화
- **비트레이트**: 96~128kbps (음질·용량 균형)
- **곡 길이**: 평균 2~3분 (반복 재생 전제 → 짧은 곡 OK)
- **CDN**: Supabase Storage + Cloudflare 자동 캐시
- **pre-load**: `metadata only` (실제 다운로드는 사용자 첫 탭 시점)
- **데이터 절약 모드**: 사용자가 OFF 가능
- **MVP**: ✅ 포함 | **우선순위**: P1
