# 인증 — 화면 기능 명세서

> 경로: `/login` · `/signup` · `/verify-phone`  
> 인증: 불필요 (비로그인 전용, 로그인 상태에서 접근 시 `/my`로 리디렉트)  
> 버전: v1.0 | 작성일: 2026-04-23  
> 관련 정책: `02-정책/인증-정책.md`

---

## 화면 흐름

```
[홈 CTA 클릭]
      ↓
[/login] ─── Google 로그인 ──→ [/verify-phone] ──→ [/create or /my]
           └── Facebook 로그인 ↗
```

- 소셜 로그인 완료 후 전화번호 인증이 없으면 반드시 `/verify-phone`으로 이동
- 전화번호 인증 완료 계정은 소셜 로그인 후 바로 서비스 진입

## 1. 로그인 화면 (`/login`)

### 화면 구조

```
┌─────────────────────────────────┐
│  [로고]                          │
│                                  │
│  "로그인하고 청첩장 만들기"        │
│                                  │
│  [Google로 계속하기]  ← 버튼     │
│  [Facebook으로 계속하기]  ← 버튼 │
│                                  │
│  ─── 또는 ───                   │
│                                  │
│  "회원가입" 링크 → /signup       │
│                                  │
│  이용약관 동의 안내 텍스트        │
└─────────────────────────────────┘
```

### 기능 상세

#### 1-1. Google 로그인 버튼
- **작동**: 탭 → Google OAuth 팝업 또는 리디렉트 → Supabase Auth 처리
- **성공 시**: 전화 인증 완료 계정 → 이전 페이지 또는 `/my`로 이동 / 미인증 → `/verify-phone`
- **실패 시**: "로그인에 실패했습니다. 다시 시도해주세요." 토스트 메시지
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 1-2. Facebook 로그인 버튼
- **작동**: 탭 → Facebook OAuth 리디렉트 → Supabase Auth 처리
- **성공/실패**: Google 로그인과 동일
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 1-3. 약관 동의 안내
- **위치**: 소셜 로그인 버튼 하단
- **문구**: "계속하면 이용약관 및 개인정보처리방침에 동의하는 것으로 간주합니다."
- **링크**: 이용약관 · 개인정보처리방침 → `/support/terms`
- **MVP**: ✅ 포함 | **우선순위**: P1

## 2. 전화번호 인증 화면 (`/verify-phone`)

> Decree 147/2024 법령 준수를 위한 필수 단계

### 화면 구조

```
┌─────────────────────────────────┐
│  [뒤로가기]                      │
│                                  │
│  "전화번호 인증"                  │
│  "베트남 법령에 따라 본인 확인이  │
│   필요합니다"                    │
│                                  │
│  [+84] [전화번호 입력]           │
│                                  │
│  [인증번호 받기] ← 버튼          │
│                                  │
│  ─ (인증번호 발송 후 표시) ─     │
│                                  │
│  [인증번호 6자리 입력]           │
│  재발송 (60초 타이머)            │
│  [확인] ← 버튼                   │
└─────────────────────────────────┘
```

### 기능 상세

#### 2-1. 국가 코드 선택
- **기본값**: +84 (베트남)
- **MVP에서는 +84 고정** (다국가 지원은 v2)
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 2-2. 전화번호 입력
- **형식**: 숫자만 입력, 최대 10자리
- **유효성 검사**: 빈 값, 10자리 미만 시 버튼 비활성화
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 2-3. 인증번호 받기 버튼
- **작동**: 탭 → Supabase Auth SMS OTP 발송
- **성공 시**: 인증번호 입력 필드 표시 + 60초 재발송 타이머 시작
- **실패 시**: "SMS 발송에 실패했습니다. 잠시 후 다시 시도해주세요."
- **재발송 제한**: 60초 대기 후 재발송 가능, 최대 5회 시도
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 2-4. 인증번호 입력
- **형식**: 숫자 6자리, 자동 포커스
- **자동 확인**: 6자리 입력 완료 시 자동으로 확인 처리
- **MVP**: ✅ 포함 | **우선순위**: P1

#### 2-5. 확인 버튼
- **성공 시**: `/create` 또는 원래 이동하려던 페이지로 이동
- **실패 시**: "인증번호가 올바르지 않습니다." (남은 시도 횟수 표시)
- **5회 초과**: "10분 후 다시 시도해주세요."
- **MVP**: ✅ 포함 | **우선순위**: P1

## 3. 에러 상태 처리

| 상황 | 메시지 | 동작 |
|---|---|---|
| 소셜 로그인 실패 | "로그인에 실패했습니다. 다시 시도해주세요." | 로그인 화면 유지 |
| SMS 발송 실패 | "SMS 발송에 실패했습니다." | 재시도 버튼 표시 |
| 인증번호 불일치 | "인증번호가 올바르지 않습니다. (N회 남음)" | 입력 초기화 |
| 인증번호 만료 | "인증번호가 만료되었습니다. 다시 받아주세요." | 재발송 버튼 활성화 |
| 5회 초과 | "잠시 후 다시 시도해주세요. (10분)" | 버튼 비활성화 + 타이머 |
