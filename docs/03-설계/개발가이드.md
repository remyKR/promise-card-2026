## 1. 프로젝트 개요

베트남 현지 맞춤형 모바일 청첩장 서비스.  
베트남인 커플이 스마트폰으로 청첩장을 직접 만들고 Zalo·Facebook으로 하객에게 공유할 수 있다.

- **서비스 언어**: 베트남어 (UI), 한국어 (주석·문서)
- **타겟 디바이스**: 모바일 우선 (Android / iOS 웹뷰)
- **개발 방식**: 바이브 코딩 (Figma 디자인 → Claude Code 구현)

---

## 2. 기술 스택

| 역할 | 기술 | 버전 |
|---|---|---|
| 프레임워크 | Next.js App Router | ^15 |
| 언어 | TypeScript | ^5 (strict) |
| 스타일 | Tailwind CSS | ^4 |
| 백엔드/DB | Supabase (PostgreSQL + Auth + Storage) | ^2 |
| 패키지 매니저 | pnpm | 10.x |
| 배포 | Vercel | — |

**추가 금지 목록**

- 상태관리: Redux, MobX (Zustand만 허용, 필요 시 논의 후 추가)
- 스타일: Styled Components, CSS Modules (Tailwind만 사용)
- HTTP 클라이언트: axios (Next.js fetch 또는 Supabase client 사용)

## 4. 폴더 구조

```
web/src/
├── app/                  # 페이지 라우트 (로직 최소화)
│   ├── (auth)/           # 로그인/회원가입 라우트 그룹
│   │   ├── login/page.tsx
│   │   └── verify-phone/page.tsx
│   ├── card/[code]/      # 공개 청첩장 열람
│   ├── create/           # 청첩장 에디터
│   ├── my/               # 마이페이지
│   │   ├── page.tsx
│   │   ├── cards/page.tsx
│   │   └── account/page.tsx
│   └── api/              # API Route Handlers
├── components/
│   ├── ui/               # 공통 UI (Button, Input, Modal, DatePicker...)
│   └── features/         # 도메인 컴포넌트
│       ├── editor/       # 청첩장 에디터 패널
│       └── card/         # 청첩장 렌더링
├── lib/
│   └── supabase/
│       ├── server.ts     # Server Component / Server Action용
│       └── client.ts     # 'use client' 컴포넌트용
├── hooks/                # 커스텀 훅 (use 접두사)
└── types/                # TypeScript 타입 정의
```

> 페이지 파일에는 UI 렌더링만 둔다. 데이터 fetch·비즈니스 로직은 Server Action 또는 `lib/`에 분리한다.

## 5. 네이밍 규칙

| 대상 | 규칙 | 예시 |
|---|---|---|
| 컴포넌트 파일 | PascalCase | `InvitationCard.tsx` |

## 6. 코드 품질 규칙

### TypeScript

- `any` 금지 — 타입을 모를 때 `unknown` 사용 후 타입 좁히기
- 모든 함수 반환 타입 명시
- `!` (non-null assertion) 남용 금지 — null 체크 후 사용

### React / Next.js

- **Server Component 우선** — 클라이언트 상태(이벤트 핸들러, useState)가 있을 때만 `'use client'`
- `useEffect` 안에서 fetch 금지 — Server Component 또는 Server Action 사용
- 이미지: `next/image` 사용 (최적화 자동 적용)
- 링크: `next/link` 사용

### Supabase

- DB 조작은 Server Component / Server Action에서만 (클라이언트에서 직접 쿼리 금지)
- RLS 활성화 전제로 코드 작성 (`service_role` 키는 서버에서만)
- 결과는 항상 error 체크:

```typescript
const { data, error } = await supabase.from('invitation_cards').select()
if (error) {
  console.error('청첩장 조회 실패:', error.message)
  return null
}
```

## 7. Figma → Tailwind 매핑

디자이너가 Figma에서 사용하는 값을 Tailwind 클래스로 변환하는 기준표.

| Figma 속성 | Tailwind |
|---|---|
| Auto Layout (horizontal) | `flex flex-row` |
| Auto Layout (vertical) | `flex flex-col` |
| Gap 4px | `gap-1` |
| Gap 8px | `gap-2` |
| Gap 12px | `gap-3` |
| Gap 16px | `gap-4` |
| Gap 24px | `gap-6` |
| Padding 8px | `p-2` |
| Padding 12px | `p-3` |
| Padding 16px | `p-4` |
| Padding 24px | `p-6` |
| Fill Container | `w-full` |
| Hug Contents | (기본값, 별도 클래스 불필요) |
| Corner Radius 4px | `rounded` |
| Corner Radius 8px | `rounded-lg` |
| Corner Radius 16px | `rounded-2xl` |
| Corner Radius 9999px | `rounded-full` |
| Fill (색상) | `bg-[#HEX]` 또는 `bg-primary` 등 디자인 토큰 변수 |
| Text color | `text-[#HEX]` 또는 `text-foreground` 등 |
| Opacity 50% | `opacity-50` |
| Shadow (md) | `shadow-md` |

## 8. 컴포넌트 설계 원칙

- `components/ui/` — 도메인 무관, 재사용 가능한 기본 요소 (Button, Input, Card, Modal...)
- `components/features/` — 특정 도메인 컴포넌트 (EditorPanel, InvitationPreview...)
- 한 컴포넌트 = 한 파일. 200줄 초과 시 분리 검토
- Props 타입은 파일 상단에 인터페이스로 명시

```typescript
interface ButtonProps {
  variant: 'primary' | 'secondary' | 'ghost'
  size?: 'sm' | 'md' | 'lg'
  disabled?: boolean
  onClick?: () => void
  children: React.ReactNode
}
```
