## 1. 시스템 아키텍처

```
[사용자 (모바일 우선)]
    ↓
[Vercel Edge Network]
    ↓
[Next.js 15 App Router]
    ├── Server Components (기본)
    ├── Client Components ('use client' 명시 시만)
    └── API Routes (Server Actions)
          ↓
    [Supabase]
     ├── Auth (Google, Facebook OAuth)
     ├── Database (PostgreSQL + RLS)
     └── Storage (이미지)
```

## 4. API 설계

| Method | Endpoint | 인증 | 설명 |
|---|---|---|---|
| GET | `/api/cards` | 필요 | 내 청첩장 목록 |

## 5. 인증 설계

### MVP 로그인 수단
| 수단 | MVP | 이유 |
|---|---|---|
| **Google** | ✅ | 글로벌 표준, Supabase 기본 지원 |
| **Facebook** | ✅ | 베트남 7,270만 사용자 |
| **Zalo** | ✅ | 베트남 7,300만 사용자 — MVP 포함으로 변경 |
| **전화번호 OTP** | ✅ | Decree 147/2024 법령 준수 |

### 인증 상태
| 상태 | 설명 | 접근 범위 |
|---|---|---|
| `guest` | 비로그인 | 공개 청첩장만 |
| `pending_phone` | SNS 인증 완료, 전화 미인증 | 전화 인증 페이지만 |
| `verified` | 전화 인증 완료 | 전체 서비스 |

### 회원가입 플로우
```
Google / Facebook 로그인
    ↓
전화번호 입력 (+84 베트남)
    ↓
OTP 인증 (SMS)
    ↓
가입 완료 → 청첩장 만들기
```
